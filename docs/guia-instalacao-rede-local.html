1. Visao Geral da Arquitetura

O Organizador OPECVAR roda como um unico servidor que serve tanto a API (backend) quanto a interface web (frontend). Qualquer computador na mesma rede acessa o sistema pelo navegador.

Arquitetura em Rede Local

[PC Servidor - Windows]
└— PostgreSQL (porta 5432)
└— Node.js / Express (porta 3001)
    └— API REST (/api/*)
    └— Frontend estatico (/*)

↓  ↓  ↓

[PC 1 - Navegador]    [PC 2 - Navegador]    [PC 3 - Navegador]
http://192.168.x.x:3001

3. Instalar e Configurar o PostgreSQL

1 Baixar o PostgreSQL

Acesse https://www.postgresql.org/download/windows/ e baixe o instalador para Windows.
Escolha a versao mais recente (16.x ou superior).

2 Instalar

Execute o instalador e siga os passos:

  • Marque todos os componentes (PostgreSQL Server, pgAdmin, Command Line Tools)
  • Diretorio de dados: mantenha o padrao
  • Defina a senha do usuario postgres — anote essa senha, voce vai precisar dela!
  • Porta: mantenha 5432 (padrao)
  • Locale: mantenha o padrao
3 Criar o banco de dados

Abra o pgAdmin (instalado junto com o PostgreSQL) ou use o terminal psql:

-- Via psql (abra o "SQL Shell (psql)" no menu iniciar)
-- Conecte como usuario postgres com a senha que voce definiu

CREATE DATABASE opecvar;

Ou pelo pgAdmin: clique com botao direito em "Databases" → "Create" → "Database" → Nome: opecvar → Save.

4. Instalar o Node.js

1 Baixar e instalar

Acesse https://nodejs.org e baixe a versao LTS (recomendada).

Execute o instalador com todas as opcoes padrao. Marque a opcao de adicionar ao PATH se perguntado.

2 Verificar a instalacao

Abra o Prompt de Comando ou PowerShell e execute:

node --version
npm --version
Deve exibir algo como v20.x.x e 10.x.x

5. Copiar o Projeto para a Maquina

Escolha uma das opcoes abaixo:

Opcao A: Copiar a pasta manualmente

Copie toda a pasta Organizador_OPECVAR para um local na maquina servidor, por exemplo:

C:\Sistemas\Organizador_OPECVAR

Opcao B: Clonar via Git

# No Prompt de Comando ou PowerShell:
cd C:\Sistemas
git clone <URL_DO_REPOSITORIO> Organizador_OPECVAR
Nao coloque o projeto em pastas com espacos ou caracteres especiais no caminho (ex: "Meus Documentos").

6. Instalar Dependencias

1 Abrir terminal na pasta do projeto 
cd C:\Sistemas\Organizador_OPECVAR
2 Instalar todos os pacotes 
npm install

Este comando instala as dependencias do backend e do frontend automaticamente (monorepo com workspaces).

Pode levar alguns minutos na primeira vez. Aguarde ate finalizar sem erros.
3 Gerar o Prisma Client 
npm run prisma:generate --workspace=backend

7. Configurar Variaveis de Ambiente

1 Criar/Editar o arquivo .env

Na pasta backend\, crie ou edite o arquivo .env com o seguinte conteudo:

# Conexao com o banco PostgreSQL
# Formato: postgresql://USUARIO:SENHA@HOST:PORTA/NOME_BANCO
DATABASE_URL="postgresql://postgres:SUA_SENHA_AQUI@localhost:5432/opecvar"

# Segredo para tokens JWT (troque para algo unico e longo!)
JWT_SECRET="minha-chave-secreta-com-pelo-menos-32-caracteres-aqui"

# Porta do servidor
PORT=3001

# Modo de execucao
NODE_ENV=production

# Origens permitidas (opcional - IPs de rede local sao aceitos automaticamente)
# So precisa preencher se houver acesso de fora da rede privada
# ALLOWED_ORIGINS="http://exemplo.com"
Substitua:
  • SUA_SENHA_AQUI pela senha do PostgreSQL definida na instalacao
  • 192.168.X.X pelo IP real da maquina servidor (veja como descobrir na secao 12)
  • JWT_SECRET por uma string aleatoria e longa para seguranca
Se muitos computadores acessam o sistema (IPs variados), voce pode listar varios IPs separados por virgula em ALLOWED_ORIGINS, ou, para facilitar em ambiente interno, adicionar todos os IPs da sua faixa de rede (ex: http://192.168.1.100:3001,http://192.168.1.101:3001,...).

8. Criar o Banco de Dados e Popular

1 Rodar as migrations (criar tabelas) 
cd C:\Sistemas\Organizador_OPECVAR
npx --workspace=backend prisma migrate deploy
O comando migrate deploy aplica todas as migrations existentes no banco de producao.
2 Popular com dados iniciais (seed) 
npm run prisma:seed --workspace=backend
Cria o usuario administrador e analistas de teste no banco.

9. Buildar o Frontend

1 Gerar os arquivos estaticos 
npm run build --workspace=frontend

Isso gera a pasta frontend\dist\ com o HTML, CSS e JS otimizados.

O backend serve esses arquivos automaticamente em modo producao.

10. Iniciar o Servidor

1 Buildar o backend (compilar TypeScript) 
npm run build --workspace=backend
2 Iniciar o servidor 
npm run start --workspace=backend
Voce vera a mensagem:
Server running on http://0.0.0.0:3001 [production]

O servidor ja esta rodando e acessivel na rede!
Mantenha essa janela do terminal aberta. Se fechar, o servidor para. Veja a secao 13 para rodar como servico em segundo plano.

11. Liberar no Firewall do Windows

O Firewall do Windows pode bloquear conexoes de outros computadores. E necessario criar uma regra de entrada.

1 Abrir o Firewall do Windows com Seguranca Avancada

Pressione Win + R, digite wf.msc e pressione Enter.

2 Criar regra de entrada
  1. Clique em "Regras de Entrada" no painel esquerdo
  2. Clique em "Nova Regra..." no painel direito
  3. Selecione "Porta" → Avançar
  4. Selecione "TCP" e em "Portas locais especificas" digite: 3001 → Avançar
  5. Selecione "Permitir a conexao" → Avançar
  6. Marque "Dominio", "Particular" e "Publico" → Avançar
  7. Nome: Organizador OPECVAR (porta 3001) → Concluir

Alternativa: Via PowerShell (Administrador)

# Abra o PowerShell como Administrador e execute:
New-NetFirewallRule -DisplayName "Organizador OPECVAR (porta 3001)" `
  -Direction Inbound -Protocol TCP -LocalPort 3001 -Action Allow

12. Acessar de Outros Computadores

1 Descobrir o IP da maquina servidor

No Prompt de Comando da maquina servidor, execute:

ipconfig

Procure por "Endereco IPv4" na interface de rede ativa (Wi-Fi ou Ethernet).
Sera algo como: 192.168.1.100

2 Acessar pelo navegador

Em qualquer computador da mesma rede, abra o navegador e acesse:

http://192.168.1.100:3001

(substitua pelo IP real da maquina servidor)

A tela de login do Organizador OPECVAR deve aparecer!
3 Atualizar ALLOWED_ORIGINS (se necessario)

Se aparecer erro de CORS, adicione o IP de origem em ALLOWED_ORIGINS no arquivo .env.

Uma alternativa pratica para redes internas: liste os IPs de todos os computadores que acessarao, ou considere usar NODE_ENV=development se a seguranca da rede interna ja e controlada.

Dica para facilitar o acesso: Se possivel, configure um IP fixo na maquina servidor para que o endereco nao mude ao reiniciar o roteador.

13. Rodar como Servico em Segundo Plano (Opcional)

Para que o servidor inicie automaticamente e nao dependa de uma janela aberta, use o pm2:

1 Instalar o pm2 globalmente 
npm install -g pm2
2 Iniciar o servidor com pm2 
cd C:\Sistemas\Organizador_OPECVAR\backend
pm2 start dist/index.js --name opecvar
3 Comandos uteis do pm2 
# Ver status do servidor
pm2 status

# Ver logs em tempo real
pm2 logs opecvar

# Reiniciar o servidor
pm2 restart opecvar

# Parar o servidor
pm2 stop opecvar
4 Iniciar automaticamente com o Windows (opcional) 
# Salvar a lista de processos para reinicio automatico
pm2 save

# Configurar para iniciar no boot (Windows)
pm2-startup install
Siga as instrucoes que o pm2 exibir para completar a configuracao no Windows.

14. Solucao de Problemas

ProblemaSolucao
ECONNREFUSED ao conectar no banco Verifique se o PostgreSQL esta rodando (Servicos do Windows → postgresql). Confira a senha no .env.
Pagina nao carrega de outro PC Verifique se a regra do Firewall foi criada (secao 11). Teste com ping IP_SERVIDOR do outro PC.
Erro de CORS no navegador Adicione o IP de origem (do PC que esta acessando) em ALLOWED_ORIGINS no .env e reinicie o servidor.
JWT_SECRET deve ter ao menos 32 caracteres Troque o JWT_SECRET no .env por uma string com 32+ caracteres.
Cannot find module Rode npm install novamente na raiz do projeto.
Tela branca / erro 404 Verifique se o build do frontend foi feito (frontend\dist\ deve existir). Rode npm run build --workspace=frontend.
Porta 3001 ja em uso Troque o PORT no .env para outra porta (ex: 3002) e atualize a regra do Firewall.
